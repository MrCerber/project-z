# Project Z — Промежуточный итог v4

> Кодовое название: **Project Z** (внутреннее использование).
> Финальное название будет определено позже.

---

## Видение

Глобальная коммуникационная платформа с открытым протоколом.
Компания израильская — продукт универсальный, для всего мира.
Как Waze, Mobileye, Check Point — израильское происхождение, глобальное использование.

---

## Основные ценности

- **Свобода слова** — до чёткой человеческой красной линии (не политической)
- **Структурная прозрачность** — политика, реклама, боты всегда маркируются
- **Без капитуляции** — никакого подчинения политическому давлению
- **Твои данные — твои** — полное право на удаление
- **Личный аудит-лог** — каждый пользователь видит все действия системы, связанные с ним
- **Прозрачность хранения** — пользователь всегда знает, где данные и кто за них отвечает
- **Нет скрытой рекламы** — любой спонсированный контент обязан быть помечен
- **Нет политической рекламы** — любой политический контент всегда виден с пометкой

---

## Этика и красные линии

### Философия
Красные линии основаны не на политических трендах, а на базовых человеческих нормах — тех, которые разделяет любой нормальный человек независимо от взглядов. Платформа не навязывает идеологию — она защищает людей.

### Запрещено абсолютно
- Контент сексуального характера с несовершеннолетними (до 18 лет)
- Любые фотографии/видео несовершеннолетних без явного согласия родителей
- Планирование терроризма
- Торговля людьми

### Прозрачность
- Любая политическая активность — обязательная пометка, видна всем
- Любая платная реклама — обязательная пометка, нет скрытых интеграций
- Боты — всегда помечены как боты, не могут притворяться людьми

### Что всегда разрешено
Любые мнения, идеологии, политические взгляды — если они не нарушают красные линии выше.

## Политика модерации

```
При жалобе:
└── Бот запрашивает доступ к серверу
    ├── Доступ дан → проверка → нарушение найдено → удаление из Registry
    └── Отказ или молчание в течение X часов → немедленное удаление из Registry

После удаления:
├── Снапшот доказательств (зашифрован, только для властей)
├── Жалоба в соответствующие органы (Интерпол, ФБР, Европол, местная полиция)
└── Передача всех доказательств — дальше это зона ответственности властей

Зона ответственности:
├── Закрытый приватный Self Hosted сервер → не наша зона
└── Публичный или полупубличный сервер → подчиняется Registry
```

**Принцип симметрии — применяется ко всем без исключения:**
- Преступники → удаление ✅
- Экстремисты → удаление ✅
- Правительство без судебного ордера → отказ ✅
- Любое политическое давление → отказ ✅
- Корпоративное давление → отказ ✅

## Архитектура

### Глобальный Cloud Registry
- Хранит минимум: имя, аватар, список друзей, список серверов
- Реестр Self Hosted серверов (имя, домен/IP, публичный ключ)
- Управление только красными линиями

### Типы серверов

| | Cloud | Self Hosted |
|---|---|---|
| Данные | У нас | У владельца |
| Функционал | Полный | Полный (идентичный) |
| Тип | Выбирается при создании — навсегда | Выбирается при создании — навсегда |
| Переход Cloud→SH | Невозможен | — |
| Data Residency | Нет | Да (владелец выбирает страну) |
| Ответственность за данные | Project Z | Владелец |

> **Важно:** Cloud сервер не может стать Self Hosted — это нарушило бы доверие всех участников сервера, которые не давали согласия на переход.

### Self Hosted — варианты развёртывания
- VPS, Docker, домашний компьютер, готовый хостинг-пакет от Project Z
- Для работы — обязательная регистрация и привязка к аккаунту владельца в Registry
- Владелец несёт полную ответственность за данные, хранящиеся на его сервере

### Уведомления пользователям
```
Всегда видно:
├── 🟢 Cloud — данные хранятся у Project Z
└── 🔵 Self Hosted — данные у [имя владельца], Project Z не несёт ответственности

Критические события (обязательное уведомление):
├── ⚠️ Сервер недоступен длительное время
├── 🔴 Смена владельца сервера
└── ℹ️ Любое изменение, касающееся хранения данных пользователя
```

## Протокол и связь

| Тип | Технология |
|---|---|
| Текст | Собственный открытый протокол |
| Голос / Видео / Стриминг | WebTransport P2P |
| Шифрование соединения | TLS 1.3 (встроен в WebTransport) |
| Сигнализация | STUN/TURN сервер |

### Открытый протокол
Публично задокументирован — любой может построить совместимый клиент или сервер.
Официальный клиент Project Z всегда будет лучшим — как Chrome для HTTP.
Открытый протокол = нельзя заблокировать полностью (актуально для России и других стран с цензурой).

### Шифрование

```
Публичные каналы
└── TLS 1.3 (сервер может читать для модерации)
    └── Audit log фиксирует каждый доступ

DM и приватные каналы
└── E2E полное шифрование
    ├── Текст — RSA шифрование ключом получателя
    └── Файлы/изображения — AES одноразовый ключ + RSA для ключа

Голос/видео:
├── 2 человека → E2E P2P прямой (WebTransport)
├── 3-8 человек → E2E Mesh P2P (SRTP)
└── 9+ человек → SFU сервер (SRTP, зашифровано, сервер не читает)

При смене состава канала:
└── Автоматический переход + новые ключи (пользователь не замечает)
```

### Шифрование файлов
```
1. Создаётся одноразовый AES-256 ключ
2. Файл шифруется через AES (быстро)
3. AES ключ шифруется RSA ключом получателя
4. Оба отправляются вместе
5. Сервер видит только зашифрованные данные
```

### Структура сообщения
```json
{
  "id": "msg_018e4c5b2d3a7abc9def123456789abc",
  "type": "message",
  "subtype": "message | announcement | system | reply | pinned | bot",
  "channel_id": "chn_...",
  "author_id": "usr_...",
  "content": "зашифровано",
  "timestamp": "2026-04-19T15:30:00Z",
  "edited_at": null,
  "reply_to": null,
  "attachments": [],
  "reactions": [],
  "encrypted": true
}
```

## Система ID

### Формат
```
В коде:       тип_UUID-v7
В интерфейсе: тип.UUID-v7

usr_  → пользователь
srv_  → сервер
msg_  → сообщение
chn_  → канал
bot_  → бот
rol_  → роль
att_  → файл/изображение
tkn_  → тикет
```

### Зачем UUID v7
- Содержит timestamp → можно сортировать по времени создания
- Глобально уникален → нельзя угадать следующий ID
- Стандарт → не изобретаем велосипед

### Ключи пользователя
```
При регистрации автоматически создаётся:
├── Публичный ключ RSA — хранится на сервере, виден всем
├── Приватный ключ RSA — только у пользователя, нигде не хранится
└── Fingerprint — короткий уникальный идентификатор
    └── Пример: A3F9-2K8P-X7QR-M4NT
    └── Используется для верификации при E2E общении
```

## Секреты (Secrets)

```
pzu_ → User Session Token (30 дней, автообновление)
pzs_ → Server Secret для Self Hosted (без срока, ручная отмена)
pzb_ → Bot Secret (без срока, ручная отмена)
pzr_ → Registry Secret (внутренний)
```

**Правила:**
- Создаётся один раз, показывается пользователю один раз
- Утерян → создаётся новый, старый аннулируется немедленно
- Компрометация → отмена + уведомление + запись в аудит-лог

## Клиент

Один глобальный клиент на **Tauri** (не Electron — легче, быстрее, меньше RAM).

### Три режима отображения
```
Full Mode    — полноэкранный (стандартный Discord-подобный)
Sidebar Mode — выдвигается сбоку, узкая панель
Mini Mode    — минималистичный, всегда на экране
             (вдохновлён Winamp / AIMP)
```

### Персонализация — вдохновлено MySpace
Каждый профиль и сервер могут выглядеть уникально — как в эпоху MySpace, но без ущерба для стабильности клиента.

```
Профиль пользователя:
├── Фоновое изображение
├── Цветовая схема
├── Markdown описание
└── CSS Sandboxed — только в рамках карточки профиля
    ├── Разрешено: цвета, шрифты, фоны, размеры, border, shadow
    ├── Запрещено: position fixed/absolute, z-index, external url()
    └── HTML полный — запрещён (XSS риск)

Страница сервера:
├── Баннер + логотип
├── Markdown описание
└── CSS Sandboxed — только в рамках about секции
```

## Каналы и структура сервера

### Типы каналов
```
💬 Текстовый    — обычный чат, markdown поддержка
📢 Объявления   — только администратор пишет, все читают
🎫 Тикеты       — каждый пользователь видит только свой тикет
🔊 Голосовой    — голос/видео/стриминг через WebTransport P2P
```

### Эмодзи каналов
- Каждый тип канала имеет стандартный эмодзи по умолчанию
- Пользователь может изменить или отключить
- Эмодзи — отдельный элемент интерфейса, не часть названия канала
- Название канала остаётся чистым (без эмодзи в тексте, как в Discord)

## Роли и права

### Шаблоны (создаются при создании сервера)
```
👑 Owner  — полный контроль, не передаётся в v1.0
🛡️ Admin  — управление сервером
🔧 Mod    — управление контентом и пользователями
👤 Member — обычный пользователь
⚙️ Custom — настраивается вручную на базе любого шаблона
```

### Категории прав
```
🏠 Сервер       — настройки, роли, каналы, просмотр логов
👥 Пользователи — kick, ban, mute, управление никнеймами
💬 Контент      — отправка, удаление чужих, закрепление сообщений
🤖 Боты         — использование команд, управление ботами
🔊 Голос        — вход в канал, говорить, mute других, kick из канала
```

### Ограничения создания серверов
```
Обычный пользователь → до 3 серверов
Верифицированный     → больше (TBD)
Премиум              → без ограничений
```
> Ограничение защищает от спама серверов и злоупотреблений.

## Боты

### Жизненный цикл
```
Разработка
└── Доступен только в DM разработчика

Приватное тестирование
└── Только серверы разработчика (до X серверов)

Publish Request
└── Команда Project Z проверяет поведение и права

Публичный
└── Доступен всем серверам
    ├── Оценки и отзывы от сообщества
    └── Нарушение → немедленное удаление
```

### Обязательные требования к каждому боту
```
├── 🤖 Метка BOT — всегда видна
├── 👤 Имя разработчика / компании — всегда видно
├── ✅ Verified — после проверки командой
└── 📋 Страница с полной информацией о правах и действиях
```

### Прозрачность прав
При добавлении бота пользователь видит точно:
- Какие каналы бот читает
- Куда бот отправляет данные
- Хранит ли бот что-то локально

Все действия бота → в аудит-лог пользователя.
**Бот не может притворяться человеком — никогда.**
